# REDUCE
The data kept in the **Warp10 platform** are stored in **Geo Time Series®**. Sometime you want to group those **GTS** according to some *labels* and then apply a function to generate a new aggregated **GTS**. For example group **GTS** according to their `area` *label* and compute the average temperature per `area`. Here enters the `REDUCE`, that provides the tooling for grouping **GTS** classes, applying a *reducer* to each *partition* fro each tick and then storing a *reduced* **GTS**.

A **GTS** is build for each *partition* through a *reducer* function. A *reduced* list of **GTS** is characterized by a list of *labels* which define *partitions* and a *reducer* function applied on each tick of each *partition*.

spans a certain number of microseconds called the *bucketspan*, ending at a tick called the *bucketend*. A *bucketized* geo time serie is characterized by its *bucketspan*, its *bucketcount* (the number of *buckets* in the *bucketized* GTS) and the *bucketend* of the most recent *bucket*, called *lastbucket*.

The `REDUCE` framework is used to convert a list of **Geo Time Series®** into a reduced list. The reduce process collects the measurements of the **GTS** and apply a *reducer* function for each tick within each partition.

The `REDUCE` framework comes with a number of *reducer* which implement very common aggregation functions such as `SUM`, `MIN`, `MAX`, `MEAN`, `COUNT`, etc.
